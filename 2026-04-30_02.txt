FIX 1 — Bug: import Decimal path incorrecto (Prisma 7)
  Archivo : src/lib/services/planilla.service.ts
  Cambio  : `import { Decimal } from '@prisma/client/runtime/library'` →
            `import { Decimal } from '@prisma/client/runtime/client'`
  Causa   : En Prisma 7 el sub-path `/runtime/library` no existe; el correcto
            es `/runtime/client`.
  Impacto : El módulo no podía ser importado en Vitest — bloqueaba todos los
            tests de integración. Tests unitarios previos no lo afectaban
            porque no importaban planilla.service.ts.

FIX 2 — Bug: rama CONSTRUCCION_CIVIL no construía datosConstruccionCivil
  Archivo : src/lib/services/planilla.service.ts
  Cambio  : Nueva rama `else if (regimenType === 'CONSTRUCCION_CIVIL')` entre
            la rama AGRARIO y el `else` genérico.
            Lee 3 parámetros de seed según categoriaCC del contrato:
              JORNAL_CC_{OPERARIO|OFICIAL|PEON}
              BUC_CC_{OPERARIO|OFICIAL|PEON}
              MOVILIDAD_CC_LABORABLE
            Construye datosConstruccionCivil y llama a calcularPlanilla().
            Detecta trabajaEnAltura via contrato.zonaBonificacion === 'ALTURA'.
  Antes   : El else genérico llamaba calcularPlanilla() sin
            datosConstruccionCivil → lanzaba 'datosConstruccionCivil es
            requerido' en runtime para cualquier contrato CC.
  Impacto : 1 test de integración (T4) lo verifica de extremo a extremo.

DESVIACIONES RESPECTO AL BRIEF
───────────────────────────────
1. T8 (cese y liquidación) NO implementado.
   Motivo: el modelo Liquidacion existe en el schema pero no hay ningún
   método de servicio que lo cree. Implementarlo requeriría un nuevo método
   (calcularLiquidacion o similar), lo cual está fuera del alcance del brief
   que restringe: "NO modificar funciones puras del motor".

2. bonificacionAgraria (BETA) no verificable E2E.
   Motivo: el campo recibeBETA no existe en el modelo Contrato — no hay
   forma de activar BETA desde el servicio sin una migración de schema.
   Verificado mediante tests unitarios en agrario.test.ts (Lote 4).

3. Para T5 se usaron 30 días (default de PlanillaDetalle) en vez de 26.
   El modo PRORRATEADO queda verificado igualmente. La variación por días
   parciales ya está cubierta por tests unitarios del motor.
