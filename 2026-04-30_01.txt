FIX 1 — Asignación familiar construcción civil (Ley 25129 / D.S. 035-90-TR)
  Archivo : src/lib/calculations/common/asignacion-familiar.ts
  Cambio  : Eliminada multiplicación por cantidadHijos en el régimen CC.
            La Ley 25129 establece un monto fijo de 10% RMV independientemente
            del número de hijos. Confirmado por STC 02681-2023-AA (TC) y
            Casación 2630-2009-HUAURA (Corte Suprema).
  Antes   : return esConstruccionCivil
              ? round2(rmv * 0.10 * cantidadHijos)   ← incorrecto
              : round2(rmv * 0.10);
  Después : return round2(rmv * 0.10);   // siempre fijo; esConstruccionCivil = no-op
  Impacto : 1 test actualizado (valor 339 → 113)

FIX 2 — Nuevo módulo asignación escolar CC (CAPECO-FTCCP 2026, R.M. 197-2025-TR)
  Archivo : src/lib/calculations/common/asignacion-escolar.ts (NUEVO)
            src/lib/calculations/common/index.ts (export añadido)
  Cambio  : Módulo independiente. 30 jornales básicos/año por hijo en edad escolar
            (hasta 24 años incl. educación técnica/universitaria). Carácter NO
            remunerativo. Soporta modalidades ANUAL y PRORRATEO_MENSUAL.
            No integrado en construccion-civil.ts — decisión deliberada para
            mantener separación de responsabilidades.
  Impacto : 0 tests rotos; 7 tests nuevos

FIX 3 — CTS pequeña empresa (D.S. 013-2013-PRODUCE art. 42)
  Archivo : src/lib/calculations/beneficios/cts.ts
  Cambio  : Nueva función calcularCtsPequenaEmpresa(). Fórmula: 15 jornales diarios
            al año (= ½ remuneración mensual). Reutiliza CtsInput. Retorna number.
            Remuneración computable incluye AF + HE + sexto de gratificación.
  Fórmula : ctsAnual = (remComputable / 30) × 15
            total = (ctsAnual / 12) × meses + (ctsAnual / 360) × días
  Impacto : 0 tests rotos; 5 tests nuevos

FIX 4 — Gratificación MYPE pequeña empresa (D.S. 013-2013-PRODUCE art. 41, Ley 30056)
  Archivo : src/lib/calculations/regimenes/mype.ts
  Cambio  : Nueva función calcularGratificacionMypePequena(). Aplica factor 0.50 al
            resultado de calcularGratificacion() (general). Afecta gratificacionBase,
            bonificacionExtraordinaria y total proporcionalmente.
  Impacto : 0 tests rotos; 2 tests nuevos

FIX 5 — Agrario modo de pago prorrateo (Ley 31110 art. 8, D.S. 005-2021-MIDAGRI)
  Archivo : src/lib/calculations/regimenes/agrario.ts
  Archivos: src/lib/calculations/regimenes/types.ts (campo agrario? añadido)
  Cambio  : calcularPlanillaAgrario() ahora lee datos.agrario?.modoPago.
            Default = 'PRORRATEADO' (integra grat 16.66% y CTS 9.72% en la RD).
            Opción 'SEMESTRAL' conserva semántica anterior (jornal×días sin extras).
            Los % son configurables vía porcentajeGratifProrrateo / porcentajeCtsProrrateo.
  Desviación: Tests existentes de agrario.test.ts pasaron a usar
            agrario: { modoPago: 'SEMESTRAL' } explícito para preservar valores.
            index.test.ts también actualizado.
  Impacto : 0 tests rotos (fixture actualizada); 3 tests nuevos para modos de pago

FIX 6 — BETA Bonificación Especial Trabajo Agrario (Ley 31110 art. 8 inc. 2)
  Archivo : src/lib/calculations/regimenes/agrario.ts
  Archivos: src/lib/calculations/regimenes/types.ts (campo recibeBETA? añadido)
            src/lib/calculations/regimenes/types.ts (bonificacionAgraria? en ingresos)
  Cambio  : BETA = 30% RMV proporcional a días trabajados. Carácter NO remunerativo:
            se suma a totalIngresos pero no integra remuneracionBruta (base pensionable/EsSalud).
            Retornado como bonificacionAgraria?: number (campo opcional en ResultadoPlanilla).
  Fórmula : beta = round2(rmv × 0.30 × (diasTrabajados / 30))
  Impacto : 0 tests rotos; 3 tests nuevos

DESVIACIONES RESPECTO AL BRIEF
───────────────────────────────
1. Fixture agrario.test.ts actualizada a SEMESTRAL explícito.
   Motivo: el default PRORRATEADO haría fallar los 4 tests existentes que
   asumen RB = jornal × días sin prorrateo. Se añadió agrario: { modoPago: 'SEMESTRAL' }
   en lugar de actualizar los valores esperados, para mantener la trazabilidad
   del caso SEMESTRAL como caso base documentado.

2. index.test.ts también requirió el mismo ajuste en el test agrario del orquestador.
   Estaba fuera del scope del brief pero era necesario para el suite verde.

3. esConstruccionCivil se mantiene como parámetro no-op en asignacion-familiar.ts
   para evitar refactorización en cascada de 4 módulos de régimen.
